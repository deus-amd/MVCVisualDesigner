User Guide

Note: This tool is still in prototype stage, and following steps may change in future versions.

0.         In this guide, we will create a simple view and its related model definition, event handling as below figure:

In the left part of this diagram, there is a Table, and in the right part of diagram, it defines the event handling for clicking Delete button.

The round rectangle shapes represent Actions, and Action in yellow is client action, and Action in red is Server action. In above figure, Show is a client action, and DeleteFriendInfo is a server action. Client action is a JavaScript action, and Server action is a MVC action.

 

And the Joints (small circle shapes in green) represent what role a widget plays in an action, for example, Replace Content action has Of, With joints, by using these joints, we can compose something like Replace Content Of ...With ..., in above example, only Of joint is used for Replace Content action. Some joints may not have a label such as the one on Show action.

 

So for the above figure, it's easy to figure out that when click Delete button, a Confirm Dialog will pop up, and after click yes on the confirm dialog, an AJAX request will be sent to DeleteFriendInfo MVC action, and finally the content of table will be replaced by the returned partial view.

25.      Now it's time to generate code from the diagram we created in above steps.

From the diagram, what we can generate including:

1)     Partial View file (*.cshtml)

2)     View Model class definition (*.cs)

3)     JavaScript event handling (*.js)

4)     JavaScript widget initialization (*.js)

5)     Selenium (www.seleniumhq.org) Page Wrapper class (*.cs)

6)     Basic test cases written in selenium according to the event handling diagram (*.cs)

7)     JavaScript and C# validation code (not shown in this example) (*.js & *.cs)

8)     The dummy code of MVC controller file (*.cs)
